## 1. Document-scoped авторинг (Фаза 2 — главный блокер)

Сейчас есть только **использование** `$ref`/`{{expr}}`, но нет **объявления** словарей.
Все 4 демо задают тему через коллекцию переменных → без этого миграция невозможна.

- **Variable collections** → `DesignDocument.variables`. Все демо определяют коллекцию
  `theme` (modes light/dark, 12 переменных, среди них var→var alias `$color.accent`).
  Нужен: ветка экстрактора (S10), parse-in (S11), emit (S12). Десугар — в существующий
  `VariablesBlockReader`/`VariablesPatch`/`liftDocumentPatches` (готовы).
- **Component definitions** → `DesignDocument.components`/`componentSets`. Сторона
  **инстанса** готова (P8); сторона **объявления** — нет. Нужны новые kinds
  `axis`/`prop`/`combination`/`set`, десугар в `component:`-блок → `ComponentLifter`.
  Определения лифтятся в `document.components` и **исчезают из дерева страниц** — поэтому
  `emitDocument` (обходит только страницы) их не видит; нужен отдельный emit-pass (S13/S14).
  `componentSets` — производные (lossy для мульти-оси; демо single-axis).
- **Shared styles** → `DesignDocument.styles`. **Полностью новый путь** (наивысший риск):
  новый `SharedStyleBlockReader` + `StylesPatch` + `TypedBlockKind.Styles` + lift (S15/S16).
  *Ни одно из 4 демо shared styles не определяет* — но нужно для полного паритета/будущего.
- **Инфраструктура (S10):** ветка document-scope в `SemanticExtractor` — заголовок
  `Collection`/`Components`/`Styles` становится document-секцией, НЕ `cnlNode`. **Без этого
  словарные предложения станут layout-узлами с коллизиями id (риск #1).**
- **Капстоун (S18):** единый `emitDocument` = variables + components + styles + тело страниц.

## 2. Reader-gap'ы (эмиттер сейчас → `null`; закрыть до удаления)

Каждый требует правку в `engine/ir` и/или ридере + грамматику. Разведка-каталог (14 позиций):

| Gap | Сложность | Суть | IR-поле |
|-----|-----------|------|---------|
| **G-TEXTKEY** (S1) | S | `content.key` не эмитится (теряется на regen) | `DesignNodeKind.Text.content.key` |
| **G-WEIGHT** (S2) | S | вес шрифта только 700/600/300; нет `weight N` | `textStyle.fontWeight` |
| **G-ADDL** (S3) | M | `visible`/`locked` читаются, но не эмитятся; `order` — оставить неявным | `DesignNode.visible/locked/order` |
| **G-GRIDTRACKS** (S4) | M | гетерогенные треки `1fr/2fr/hug` не читаются reader'ом и не эмитятся | `DesignAutoLayout.columns/rows` (List уже готов) |
| **G-IMPLICITROWS** (S5) | M | non-1fr implicit rows недостижимы (reader всегда даёт Flex(1.0)) | `implicitRows/implicitRowMin` |
| **G-MOTIONREF** (S6) | S | ref-less inline motion (`ref=""`) → `null` | motion fallback keyframes |
| **G-STROKESTYLE** (S7) | M | `strokeStyle`-ref не читается/не мерджится/не эмитится | `DesignNode.strokeStyleId` |
| **G-STICKY** (S7) | trivial | `scroll.sticky` не читается/не эмитится | `DesignScroll.sticky` |
| **G-VARMODES** (S8) | M | per-node `variableModes` (пин мода) нет нигде в грамматике | `DesignNode.variableModes` |
| **G-OVR** (S8b/S24) | hard | prop-bound `characters`; `overrides.set` по id-пути (7 групп свойств) | `InstanceOverride.*`, Text `characters` PropRef |
| **G-WPS** (S25) | M | `weightPerSide` (per-side stroke weight) не читается/не эмитится | `DesignStrokes.weightPerSide` |
| **G-STROKEPAINT** (S26) | M | per-paint stroke opacity/blend/visible + non-solid stroke paints | `DesignStrokes.paints[*]` props |
| **G-SPANS** (S27) | hard | rich-text style-спаны (`styleRef` + inline) — **новое IR-поле** | `TextStyleRange` (нет `styleRef`) |
| **G-BIND** (S28) | hard | `$ref/{{expr}}` в literal-only слотах — поля НЕ `Bindable` (структурно невыразимо) | effects x/y/blur/spread, grid-числа, media asset/poster/focal |
| **G-MASK** (S29) | hard | `mask.source` — **нет IR-поля + новая семантика резолвера** | `DesignMask` (только type+appliesTo) |
| **G-BOOL** (S29) | hard | boolean-op children id-list — **рекомендуется отбросить** (операнды = дети поддерева) | `BooleanOperation` (только op) |

> **По D4:** G-MASK/G-BIND/G-BOOL — behavior-bearing; ни одному из 4 демо не нужны. Решить
> per-gap: закрыть (построить IR+resolver) или явно отбросить с трекингом.
