# План: миграция на CNL-only (упорядоченный, с гейтами)

Порядок шагов, выведенный из архитектурной разведки (4 read-only агента + синтез).
Каждый шаг **компилируется и держит тесты зелёными**. Полная цель — в [1-ЦЕЛЬ](1-ЦЕЛЬ.md);
статус выполнения — в [3-СДЕЛАНО](3-СДЕЛАНО.md) и [4-ОСТАЛОСЬ](4-ОСТАЛОСЬ.md).

## Глобальные конвенции (для каждого шага)

- **Gate (обязателен перед «done»):**
  `./gradlew :engine:ir:jvmTest :engine:frontend:jvmTest :shared:jvmTest :desktopApp:compileKotlin`
- **Паттерн бакета (на каждую новую фразу):** spec (`design-book/semantic-layout-markdown-i18n.md`)
  → `CnlPropertyKind` (`CnlModel.kt`) → keyword (lowercase) в `CnlVocabulary` → consumer в
  `CnlParser` (group-scoped) → desugar → `Descriptor(kind, keyword, order, ::render)` + render
  в `CnlGrammar` → round-trip-тест → gate.
- **Kotlin-гочи:** НЕ `return null` внутри `buildList`/`joinToString`/`mapNotNull`-лямбд;
  НЕ `return` в expression-body `when`-функциях; ключи vocab-мап — lowercase; рендереры
  kind-gated, возвращают `null` (не бросают) для неприменимых узлов; `Descriptor.order` —
  без осмысленных коллизий (ничьи только там, где рендереры не сосуществуют).
- **Sequential-ограничение:** любой шаг, трогающий `CnlGrammar.kt`, `CnlParser.kt`,
  `CnlVocabulary.kt`, `CnlModel.kt`, `PatchMerger.kt`, `TypedPatch.kt`, `SlmPatcher.kt`, —
  **строго по одному агенту за раз** (параллельные конфликтуют). Практически это
  сериализует почти весь план; параллелиться могут только markdown (spec/skill/README)
  и новые изолированные тест-файлы.
- **Эмиттер растёт сам:** `CnlEmitter.phrasesOf` обходит
  `CnlGrammar.descriptors.sortedBy{order}.mapNotNull{render}` — новый `Descriptor` эмитится
  автоматически.

## Решения, которые нужны от человека ПЕРЕД началом

Блокируют Фазу 2 и Фазу 6. Рекомендации даны; финальное слово за пользователем.

- **D1 — Дом для document-словарей** (блокирует всю Фазу 2). Variable collections /
  component definitions / shared styles — не узлы. Варианты: (a) **секции в теле CNL** с
  «служебными» заголовками (`# Collection <id> …`, `# Components`, `# Styles`), которые
  экстрактор трактует как document-scope, а не как layout-узлы; (b) YAML frontmatter.
  **Рекомендация: (a)** — консистентно с «CNL — единственный авторинг».
- **D2 — Зарезервированные существительные для словарных предложений.**
  `Text`/`Color`/`Effect`/`Fill`/`Grid` пересекаются с нодовыми существительными и
  keyword'ами. Варианты: строгий **context-gating** (смысл существительного меняется внутри
  секции `# Collection`/`# Styles`) vs отдельные написания. **Рекомендация: context-gating**
  через жёсткую ветку экстрактора.
- **D3 — `prop` (объявление) vs `props` (override инстанса); `variant` def vs instance.**
  Почти-омографы. **Рекомендация:** у объявления — `axis`/`prop`/`combination`/`set`; у
  инстанса — прежние `variant`/`props`.
- **D4 — Объём закрытия behavior-bearing gap'ов.** `mask.source`, boolean-op `children`
  id-list, `$ref/{{expr}}` в literal-only слотах — требуют НОВЫХ IR-полей и/или семантики
  резолвера. Раз `​```ir` удаляется, любая фича, достижимая ТОЛЬКО через него, исчезнет,
  если gap не закрыть. **Рекомендация:** G-BOOL → отказаться от id-list-формы (операнды —
  дети поддерева, CNL их уже вкладывает); G-MASK и G-BIND → закрывать только если нужны
  живой фиче/демо (ни одному из 4 демо не нужны — см. [4-ОСТАЛОСЬ](4-ОСТАЛОСЬ.md)), иначе
  отложить с трекингом.
- **D5 — Судьба `LegacyMissionOverviewFixture`** (блокирует Фазу 6). Фикстура удержана
  специально ради `​```ir`-splice и `$prop`-биндингов `visible`/`characters`. Варианты:
  (a) полностью мигрировать в CNL (нужны node-body `$prop`-фразы для `visible` и
  `characters` + закрытие G-OVR); (b) оставить минимальный ir-splice только для теста
  (противоречит «sole authoring»); (c) удалить фикстуру, перевесив её write-back/structural
  покрытие на CNL-фикстуры. **Рекомендация: (a)** если D4 закрывает G-OVR + добавляет
  node-body `$prop`; иначе **(c)**.

## ФАЗА 0 — Emitter-only gaps, блокирующие эквивалентность демо (sequential)

Внутри существующей нод-грамматики; предпосылки к точной эмиссии демо.

- **S1 — эмиссия TextKey** (i18n-ключи) — S. Добавить `Descriptor(TextKey, "key", …)` +
  `renderTextKey` (`content.key`). Сейчас ключ молча теряется при emit. → блокирует все демо с `key:`.
- **S2 — произвольный вес шрифта `weight 500`** — S. `renderFontWeight` сейчас знает только
  700/600/300; добавить fallback `weight N` (и parse). → 29 узлов в MissionOverview.
- **S3 — эмиссия `visible`/`locked`** узла — S. Kinds `Visible`/`Locked`; читаются, но не
  эмитятся. `order` — оставляем неявным (порядок из вложенности), задокументировать.
- **S4 — гетерогенные grid-треки** `columns (tracks (1fr 2fr hug))` — M (gap G-GRIDTRACKS).
  Reader: список `tracks:`; grammar: перестать `return null` при разных треках.
- **S5 — non-1fr implicit rows** `rows (auto track 2fr min N)` — M (gap G-IMPLICITROWS),
  зависит от S4. → разблокирует MissionTelemetry `rows:{auto,track:hug}`.
- **S6 — ref-less inline motion** — S. `renderMotion` сейчас `null` при пустом `ref`;
  эмитить fallback-only keyframes. → badge_dot pulse в MissionTelemetry.

## Ключевые риски

- **Фантом-ноды** — доминирующий риск всех трёх словарей: S10 должен приземлиться раньше
  любого словарного parse-шага, иначе словарные предложения станут layout-узлами с
  коллизиями id.
- **CNL компилируется косвенно:** `CnlElementBlock → SemanticExtractor.cnlNode →
  CnlParser.desugar → List<TypedEntry> → TypedBlockReader → TypedPatch → PatchMerger`.
  Новые словари переиспользуют этот путь (variables/component — готовые readers; styles —
  единственный новый reader).
- **componentSets — производные** (`ComponentLifter.rebuildSet`): эмиттер def'ов (S14)
  реконструирует оси/combination/set, lossy для мульти-оси; демо single-axis — безопасно,
  но проверить явно.
- **Silent CNL→YAML fall-through** (`SlmPatcher.kt:115`) — S30 обязан его убрать, иначе
  смешанный авторинг сохранится.
