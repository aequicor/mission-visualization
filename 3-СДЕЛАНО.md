## ✅ Готова: двусторонняя грамматика CNL (нод-уровень, P0–P10)

Это «грамматическая половина» миссии — **поставлена как PR #12**
(https://github.com/aequicor/mission-visualization/pull/12), ветка
`claude/cnl-support-format-migration-e4f518`, коммит `947a996`
«Make CNL a fully bidirectional authoring format at IR parity».

### Архитектурные краеугольные камни
- **`cnl/CnlGrammar.kt`** — реестр `Descriptor(kind, keyword, order, render)` — единый
  источник истины для парсинга И генерации. **70 дескрипторов**, рендереры kind-gated.
- **`cnl/CnlEmitter.kt`** — детерминированный IR→CNL: `emitSentence`, `emitHeadingSuffix`,
  `emitHeadingLine`, `emitSubtree`, `emitDocument` (регенерирует тело каждого экрана).
  `phrasesOf` растёт автоматически с реестром.
- **`cnl/CnlVocabulary.kt`** — English-only: существительные + **82 keyword→kind** (lowercase) +
  booleans. RU-синонимы убраны.
- **`cnl/CnlParser.kt`** — токенайзер с `( )`-структурой + рекурсивный `parseGroup`
  (`GNode`/`GLeaf`/`GGroup`); `BlockBuilder` c аккумуляторами → `toEntries()`.
- **`cnl/CnlModel.kt`** — `CnlPropertyKind` покрывает все бакеты.
- **`( )`-группы со scope-локализацией** — под-ключи резолвятся локальной таблицей бакета,
  не утекают на уровень узла (keystone против утечки per-fill `opacity`/`blend` в node style).

### `emitDocument` (генерация целого экрана)
`CnlEmitter.emitDocument(document, includeId=true)` регенерирует markdown-дерево заголовков +
одно предложение на узел для каждого экрана. Это база для миграции демо (Фаза 3). Пока
эмитит **только тело страниц**; словари уровня документа (variables/components/styles) и
frontmatter — «отдельно» (harness ещё не существует — это Фаза 2, шаг S18).

## Что это даёт
Способность «**генерировать**» (IR→CNL) на уровне узлов готова и проверена. Способность
«**понимать**» работает. «**Обновлять**» — работает хирургически для части правок (span-replace +
phrase-append), но пока падает в YAML-fallback для остального (это чинит S30). Не хватает
для полного отказа от YAML — **document-scoped словарей**, **нескольких reader-gap'ов** и
собственно **миграции демо + удаления** — см. [4-ОСТАЛОСЬ](4-ОСТАЛОСЬ.md).
