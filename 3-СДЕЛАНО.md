## ✅ Готова: двусторонняя грамматика CNL (нод-уровень, P0–P10)

Это «грамматическая половина» миссии — **поставлена как PR #12**

«Make CNL a fully bidirectional authoring format at IR parity».

### Архитектурные краеугольные камни
- **`cnl/CnlGrammar.kt`** — реестр `Descriptor(kind, keyword, order, render)` — единый
  источник истины для парсинга И генерации. **70 дескрипторов**, рендереры kind-gated.
- **`cnl/CnlEmitter.kt`** — детерминированный IR→CNL: `emitSentence`, `emitHeadingSuffix`,
  `emitHeadingLine`, `emitSubtree`, `emitDocumentBody` (регенерирует тело каждого экрана),
  `emitDocument` (standalone `.layout.md`: frontmatter + словари + тело).
  Отдельные document-scope эмиттеры: `emitVariables`, `emitComponents`, `emitStyles`.
  `phrasesOf` растёт автоматически с реестром.
- **`cnl/CnlVocabulary.kt`** — English-only: существительные + **82 keyword→kind** (lowercase) +
  booleans. RU-синонимы убраны.
- **`cnl/CnlParser.kt`** — токенайзер с `( )`-структурой + рекурсивный `parseGroup`
  (`GNode`/`GLeaf`/`GGroup`); `BlockBuilder` c аккумуляторами → `toEntries()`.
- **`cnl/CnlModel.kt`** — `CnlPropertyKind` покрывает все бакеты.
- **`( )`-группы со scope-локализацией** — под-ключи резолвятся локальной таблицей бакета,
  не утекают на уровень узла (keystone против утечки per-fill `opacity`/`blend` в node style).

### `emitDocument` / `emitDocumentBody`
`CnlEmitter.emitDocumentBody(document, includeId=true)` регенерирует markdown-дерево
заголовков + одно предложение на узел для каждого экрана.
`CnlEmitter.emitDocument(document, includeId=true)` теперь является единым SLM-emitter'ом:
YAML frontmatter (оставшийся runtime metadata слой) + CNL `Collection`/`Styles` sections +
body + lifted `Component:` definitions. Это база для миграции демо (Фаза 3).

### Adversarial-верификация Фазы 4 + hardening
Мульти-агентный workflow (4 линзы → триаж) нашёл **8 подтверждённых багов** tier-3 (silent
data-loss). Корень: у attribute-пути write-back не было fidelity-veto, а `identityFingerprint`
не покрывал appearance/behavior. Закрыто одним общим механизмом:
- **Fidelity-veto в `writeBackEdits`** (по образцу `withStructuralSource`): после recompile
  пропатченного источника узел `nodeId` должен совпасть с in-memory-намерением
  (`pageTreeIds` + `identityFingerprint`, single-source-to-single-source). Лоссовый tier-3
  re-emit (дропнутый inexpressible interaction/effect/stroke/role, срезанный line-prefix →
  reparent) отклоняется → правка остаётся in-memory, источник не портится.
- **`identityFingerprint` расширен**: `role`, `fills`, `strokes`, `effects`, `cornerRadius`,

  структурный путь — insert/duplicate узла с hidden-effect/`weightPerSide` теперь корректно

- **Root-cnlSpan фикс**: у screen-root `span=(1,lastLine)`, поэтому его CNL-owner теперь
  указывает на строку H1 (`SemanticNode.cnlSpan`), а не на frontmatter — правки H1-корня

- Регрессии: `CnlWriteBackVetoTest` (inexpressible cubic-bezier на CNL-кнопке не портит
  источник), обновлён `ComponentTypeWriteBackTest` (in-tree `type=component` → veto → in-memory).
