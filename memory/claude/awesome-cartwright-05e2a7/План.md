# План

1. Зафиксировать стартовый контекст: инструкции проекта, git status, историю ветки, исходный план и фактическую реализацию.
2. Перенести уже существующие незакоммиченные изменения веточного worktree в текущий writable-worktree без отката чужой работы.
3. Провести аудит трёх новых модулей и `:shared`-интеграции против исходного плана.
4. Запустить релевантные тесты модулей аннотаций и `:shared:jvmTest`, исправить ошибки компиляции/логики.
5. Закрыть найденные пробелы в ядре, SLM sidecar, Compose-рендерере, reducer/write-back, UI и документации.
6. Запустить полную доступную проверку сборки, затем синхронизироваться с актуальным `main`, разрешить конфликты и повторить проверки.
7. Проверить итоговый diff на случайные артефакты и границы модулей.
8. Сделать осмысленный commit, push в `claude/awesome-cartwright-05e2a7`, открыть ready PR с описанием архитектуры, UX, sidecar, тестов и ограничений.
9. Проверить mergeability/конфликты PR и зафиксировать финальный статус.
