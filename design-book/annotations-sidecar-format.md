# Формат sidecar-файла аннотаций `*.annotations.md`

[← Оглавление](README.md)

Эта спецификация фиксирует формат **review-слоя аннотаций**: комментарии
(note / issue) к экрану дизайна живут в отдельном markdown-файле рядом с
SLM-исходником экрана и **никогда не смешиваются** с дизайн-разметкой.
Реализация — модуль `:subsystems:annotations-slm`
(`AnnotationSlmParser` / `AnnotationSlmWriter` / `AnnotationSlmPatcher`,
пакет `io.aequicor.visualization.subsystems.annotations.slm`); модель —
`:subsystems:annotations` (`Annotation` / `AnnotationLayer`). Спека описывает
фактическое поведение кода и его тестов.

## Соглашение об имени файла

Каждый экран `<screen>.layout.md` может нести sidecar `<screen>.annotations.md`
(например `mission.layout.md` → `mission.annotations.md`). Sidecar — это **не
SLM**: компилятор SLM его не видит; редактор роутит такие файлы в
`AnnotationSlmParser`. Один sidecar = один `AnnotationLayer` одного экрана.

## Общая структура

Файл — markdown, где **одна секция `## ` = одна аннотация**:

```markdown
# Overview — review notes        <- преамбула: игнорируется целиком

## issue @node-abc123(8,-12) +@node-def456 [expanded] {id=ann-1, author=Alice}
Контраст текста ниже нормы,
поправить фон.
![320x200](data:image/png;base64,....)

## note @(120,340) {id=ann-2}
Свободный комментарий, откреплён от узла.
```

Любой контент **до первой** строки, начинающейся с `## ` (заголовки, проза,
комментарии ревьюера), не является аннотацией и игнорируется парсером; патчер
сохраняет его байт-в-байт. Секция тянется от своей `## `-строки до следующей
`## `-строки (исключительно) либо до конца файла — то есть **пустая строка,
разделяющая секции, принадлежит предыдущей секции** (важно для патчера, см.
ниже).

## Грамматика заголовка

```
header     = "## " kind " " anchor { " +@" nodeId } [ " [expanded]" ] " " attrs
kind       = "issue" | "note"
anchor     = "@" nodeId [ pair ]        ; привязка к узлу (+ опц. offset)
           | "@" pair                   ; свободная точка
pair       = "(" number "," number ")"
attrs      = "{id=" value [ ", author=" value ] "}"
nodeId     = 1*idChar                   ; буквы/цифры и _ - . : /
```

По порядку:

- **kind** — `issue` (замечание, жёлтый `statusWarning`, попадает в
  промпт-экспорт) или `note` (нейтральное пояснение, не экспортируется).
  Любой другой токен делает заголовок некорректным.
- **anchor** — обязателен, сразу после kind:
  - `@nodeId` — `NodeAnchor(nodeId)`: капля у top-center узла;
  - `@nodeId(dx,dy)` — `NodeAnchor` со смещением от top-center. Writer пишет
    `(dx,dy)` **только когда смещение ненулевое**;
  - `@(x,y)` — `FreePoint(x, y)`: абсолютная точка в координатах экрана.
- **`+@nodeId`** — ноль и более дополнительных ссылок на узлы (`references`),
  порядок сохраняется.
- **`[expanded]`** — опциональный флаг: авторский хинт `defaultExpanded`
  (раскрыта ли табличка по умолчанию). Runtime-состояние раскрытия — view state
  редактора и в sidecar **не пишется**.
- **`{id=..., author=...}`** — блок атрибутов. `id` — стабильный id аннотации;
  `author` — опциональный автор. Неизвестный ключ делает заголовок
  некорректным. Writer **всегда** эмитит `{id=...}` — тот же инвариант явного
  id, что у структурных правок SLM.

Числа — десятичные `Double`; канонический вывод целых значений — без `.0`
(`8`, не `8.0`); `-12.5` пишется как есть.

## Тело секции

Строки между заголовком и следующей секцией:

- **Картинка** — первая строка вида `![alt](source)` (целиком одна строка)
  становится вложенным изображением `AnnotationImage(source, width, height)`.
  `source` — data-URI либо asset-ссылка. Alt-текст вида `320x200` (десятичные
  допустимы) несёт интринсик-размер; любой другой alt (в т.ч. пустой) даёт
  размеры `0x0`. Writer пишет alt как `<width>x<height>` и ставит строку
  картинки **после** текста тела.
- **Текст** — все остальные строки; ведущие и хвостовые пустые строки
  отбрасываются (это разделители секций, не контент), внутренние переносы
  сохраняются. Тело — plain text (v1).

Ограничения v1 (документированы, не энфорсятся): строка тела не должна сама
начинаться с `## ` и не должна быть одинокой markdown-картинкой; значения
атрибутов не содержат `,` и `}`.

## Толерантность парсера (`AnnotationSlmParser`)

Парсер **никогда не роняет весь файл**; результат —
`AnnotationSlmParseResult(layer, warnings, needsRewrite)`:

- **Битая секция** (неизвестный kind, отсутствующий/кривой якорь, незакрытый
  `{...}`, неизвестный ключ атрибута, пустой id ссылки) — пропускается с
  `AnnotationSlmWarning(line, message)` (line — 1-based строка заголовка);
  остальные секции живут.
- **Дубликат явного id** — первая секция побеждает, дубликат пропускается
  с warning.
- **Секция без `{id=...}`** — получает детерминированный синтезированный id
  `ann-<1-based-номер-секции>` (при коллизии с занятым id — суффикс `-2`,
  `-3`, …) и взводит `needsRewrite = true`: вызывающая сторона должна один раз
  записать слой обратно, чтобы запинить id в исходнике.
- Пустой/пробельный файл → пустой слой; преамбула до первой `## ` — игнор.

## Writer (`AnnotationSlmWriter`)

- `write(layer)` — все аннотации по секции, разделённые одной пустой строкой.
- `renderSection(annotation)` — заголовок, строки тела (если текст непуст),
  затем строка картинки; завершается одиночным `\n` без хвостовой пустой
  строки (разделение — забота `write`/патчера).
- **Round-trip-стабильность**: `parse(write(layer)).layer == layer` по всем
  полям модели; записанный файл никогда не требует синтеза id.

## Патчер (`AnnotationSlmPatcher`) — хирургические правки

Точечные правки не пересериализуют файл: секция находится по маркеру
`{id=...}` в заголовке и сплайсится на месте; **байты всех остальных секций и
преамбулы сохраняются точно**.

- `upsertSection(source, annotation)` — заменяет секцию с совпадающим id
  свежим рендером; если такой секции нет — **дописывает** новую в конец,
  отбив ровно одной пустой строкой (каноничное разделение writer'а).
- `deleteSection(source, annotationId)` — удаляет footprint секции; неизвестный
  id — no-op (source возвращается без изменений). Удаление **хвостовой** секции
  подрезает повисшее пустое разделение до одиночного финального `\n` (пустой
  остаток → пустая строка).

Footprint секции — от её `## `-строки до следующей `## `-строки (исключительно)
либо до конца source; секция «владеет» своим пустым разделителем.

## Как sidecar живёт в редакторе

Sidecar-файлы едут в общем списке источников документа (drafts / autosave /
Save / Reset / restore — бесплатно), но вместо SLM-компиляции получают
placeholder-результат и парсятся толерантно (`annotationLayersFrom`,
`:shared/editor.domain`). Каждая правка аннотации в редакторе идёт через
`writeBackAnnotations`: чистая операция над `AnnotationLayer` + зеркальный
хирургический патч sidecar-исходника; первая аннотация экрана создаёт sidecar.
Sidecar без экрана тоже парсится — аннотации не теряются. Подробности —
`EDITOR.md`, раздел «Annotations (review layer)».

## Вне scope v1

- Freehand-ink рисование (только вложенное изображение).
- RichText в теле (тип `AnnotationBody` — задел).
- Кросс-экранные привязки/ссылки: node id — в пределах экрана своего sidecar.
