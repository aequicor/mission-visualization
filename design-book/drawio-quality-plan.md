# План: качество draw.io — рендеринг · валидация · скилл агента

Статус: план (18.07.2026). Продолжение `diagram-layout-readability-plan.md` (P0–P3 shipped)
после хотфикс-линии v1.1.0 (тихие пересечения `jumps none`, одиночный прыгун, degenerate-canvas
хардненинг, collision-aware палитра). Три направления, у каждого: текущее состояние, пакеты
работ с приоритетами и DoD. Reference-поверхности: `aeration/design` (house-ER / clean-arch /
gas-room), базлайн линта ER 5 / ca 4 / gas 4.

## Направление 1 — Рендеринг

Есть: роутинг-каскад (facing-side планирование, fan-слоты, row-fixed порты, crossing-aware A*
в два прохода + regret, спуры/петли/S-крюки до фикспоинта), тихие пересечения по умолчанию,
скругления с клампом радиуса, полный набор ER/UML-маркеров, sketch, flow-анимация, LINK/ARROW,
SVG-экспорт = канва байт-в-байт по маршрутам.

Пакеты (по видимой боли с последних скрин-ревью):

- **R1 (P0) Маркер-зоны — препятствия для чужих линий.** Endpoint-marker box (порт + длина
  маркера вдоль ребра × полуширина) входит в обструкции nudging'а и веерных слотов; заблокированный
  сдвиг пробует зеркальную полосу (механика есть). Новый линт `MarkerCovered`.
  DoD: на reference-кропах ни одна чужая линия не режет crow's foot/circle; MarkerCovered = 0.
- **R2 (P0) Швы via-хилинга.** Почти-ретрейсы с допуском (не только точные 180°): П/U-стыки пар
  co-run (staple 934→876 в house-ER), зигзаги 2–4px у портов вне окна и20-слайда.
  DoD: reference без П-швов; EdgeSpur = 0; суммарная длина маршрутов не растёт.
- **R3 (P1) Каналы между facing-таблицами.** Щель между парой таблиц — общий канал: райзеры
  пакуются channel-роутером с квотами, короткие коннекторы строка-строка идут прямыми, объезды
  через веер запрещены. DoD: гребёнка id_left ≤ 4 кроссингов; «заходы» без брейда.
- **R4 (P1) Метки v2.** Анкер не садится на порт-зону/маркеры (расширить обструкции push'а),
  приоритет свободных карманов между коридорами, плейт с адаптивной прозрачностью.
  DoD: LabelOverNode + новый LabelOverMarker = 0 на reference.
- **R5 (P1) Разгон хотспотов.** Штраф скопления в точке (spread-компонента в crossing-aware
  стоимости): пересечения расползаются по длинным пролётам вместо «расчёски» в одной точке.
  DoD: CrossingHotspot ≤ 2 на house-ER без роста общего числа кроссингов.
- **R6 (P2) Полиш паритета draw.io.** Градиентные заливки узлов, опциональная тень, dark-тема
  канвы диаграмм, LOD при отдалении (текст строк скрывается ниже порога зума), waypoint-ручки
  драга в стиле draw.io, экспорт PNG/буфер, печать.
- **R7 (P2) Перф на больших графах.** Инкрементальный ре-роут только затронутых рёбер при драге;
  кэш вееров per-pass (замер: aware 18ms/blind 8ms на 38 рёбер — приемлемо, следить при 100+).

## Направление 2 — Валидация UI/UX готовых схем

Есть: `lintDiagram` (NodeOverlap, EdgeThroughNode, AnchorBunch, EdgeOverlap, EdgeSpur,
CrossingHotspot, LabelOverNode, NodeLabelFit) — но только в тест-туле
(`SlmDiagramAuditTool`/`SlmFolderAuditTool`); в продукте находки не видны.

- **V1 (P0) Линт в продукт.** Панель «Diagram health» в инспекторе + счётчик в статус-баре;
  клик по находке = zoom к месту + подсветка виновников; severity error/warn/info; баджи на
  канве (переиспользовать паттерн annotations-overlay). DoD: все правила видны в UI, находки
  кликабельны, пересчёт на каждую правку дебаунсится.
- **V2 (P0) UX-эвристики поверх гигиены рендера.** Новые правила: off-grid позиции;
  почти-выравнивание (≤5px до общего ряда/колонки); несогласованный spacing в ранге; текст
  не влезает/обрезан (детектор к fitNodeToText); label-label коллизии; контраст текста к заливке
  (токены темы); консистентность направления потока (flowchart/BPMN); пустые контейнеры и
  висячие узлы; дубликаты видимых имён. Каждое правило проектируется сразу с авто-фиксом.
- **V3 (P1) Авто-фиксы.** Кнопка «Fix» у находки → готовый intent (Tidy/Align, slide label,
  fitNodeToText, snap-to-grid, разнос ранга) + «Fix all safe» для правок без потери авторского
  замысла. DoD: ≥70% правил V2 имеют one-click фикс; undo атомарен.
- **V4 (P1) Регрессионный контур в CI.** Reference-файлы: lint-baseline (JSON) + golden-SVG
  диффы; PR падает при «хуже baseline» (счётчики по классам + новые классы = fail). Это
  институционализация текущего ручного аудит-цикла. DoD: gradle-таска `auditBaselineCheck`
  в CI-прогоне.
- **V5 (P2) Vision-проход.** Скрипт кропов хотспотов (crop.py + headless Chrome — уже есть в
  цикле) + чек-лист «глазами» (Euler-containment и пр.); опционально LLM-vision advisory-скоринг
  читаемости (не гейт).

## Направление 3 — SKILL for AI agent (`SKILLS/SLM-diagrams.md` + `SKILLS/SLM.md`)

Есть: 399 строк грамматики CNL + основы линт-цикла; синхрон с дефолтом `jumps none` — сделан
(18.07, вместе с этим планом).

- **S1 (P0, done 18.07) Синхрон семантики рендера.** Дефолт `jumps none`, правило одиночного
  прыгуна («не автори jumps ради направления — горб всегда вверх»), collision-aware размещение
  палитры. Держать раздел скилла в одном PR с изменением дефолтов (правило репо).
- **S2 (P0) Lint-driven authoring как обязательный протокол.** Скилл предписывает цикл:
  автор → компиляция → аудит → `lint.txt` → правка по таблице «находка → приём»:
  EdgeOverlap → раздвинуть ранги/сменить порт; CrossingHotspot → перенести узел или развести
  коридоры; LabelOverNode → dx/dy в свободный карман; NodeLabelFit → размеры из таблицы ширин;
  EdgeThroughNode → не прижимать узлы к коридорам. DoD: таблица приёмов в скилле; агент не
  сдаёт схему с ненулевыми error-классами.
- **S3 (P1) Куки-бук макро-лейаута по семействам.** ER: FK-поток слева-направо, интервалы
  рангов 60/40, высоты строк; UML class: наследование вверх, композиция вниз; sequence: порядок
  lifeline по первому сообщению; BPMN: пулы/дорожки и направление. Канонические мини-примеры +
  анти-паттерны (не писать `via` руками — роутер сам и чинит их при сдвигах; не наезжать узлами;
  явные id для стабильности write-back).
- **S4 (P1) Чек-лист сдачи.** Компиляция без error; линт: ThroughNode/Overlap = 0, остальное ≤
  базлайна; «ёлочки» экранированы; ICU без `{}` в defaultText; имена/контент язык-нейтральны.
- **S5 (P2) Eval-набор скилла.** 5–10 промптов («ER подписок», «BPMN найма», «sequence логина»)
  с автопроверкой: компилится, error-классы линта нулевые, структурные ожидания (N узлов,
  контейнеры). Гоняется при каждой правке скилла (skill-creator evals).
- **S6 (P2) Аудит без тестового прогона.** Gradle-таска/CLI `auditSlm <file> <outDir>` — обёртка
  над SlmDiagramAuditTool, чтобы внешний агент гонял цикл S2 в любом окружении.

## Последовательность и связи

1. **P0-волна:** R1 + R2 (самый видимый скачок рендера) · V1 (контроль качества из тестов —
   в продукт) · S2 (агент сразу авторит по линту). S1 уже сделан.
2. **P1-волна:** R3 + R4 вместе с V2/V3 — новые UX-правила валидируют канал/метки, авто-фиксы
   опираются на те же intents; V4 закрепляет достигнутое в CI; S3/S4 переносят выводы в скилл.
3. **P2-хвост:** R5–R7, V5, S5/S6 — по мере боли.

Инвариант всех трёх направлений: **один источник истины** — правило линта = приём в скилле =
(где возможно) авто-фикс в редакторе. Новое правило без приёма в скилле и без фикса в UI не
считается закрытым.
